# Application Answers Draft

This document provides draft wording for the OpenCorporates Public Benefit Project application fields.

## Suggested Project Positioning

- Project type: independent public-benefit open-source project
- Access requested: API key
- Other datasets combined at application time: No
- Service/content produced: free to use for all

## Draft Answers

### What are the main aims of the research/project you are applying for?

This is a public-benefit open-source project that uses OpenCorporates data to help researchers, journalists, students, civil-society users and the general public investigate legal entities more easily.

The project aims to improve transparency by providing a free and openly documented workflow for company lookup, entity verification and structured research support. The intended beneficiaries are journalists, academic researchers, nonprofit investigators, students and other public-interest users.

The expected outcomes are:
1. an openly published GitHub repository,
2. reproducible documentation and workflow materials,
3. a clearly documented methodology,
4. free public access to the resulting tool/content.

### How will you use the OpenCorporates data?

We will use OpenCorporates data through the API for company lookup, entity verification, structured matching and reproducible public-interest research workflows.

OpenCorporates data will help us:
1. identify legal entities consistently,
2. link findings back to source company records,
3. improve the reproducibility of the research process,
4. publish a free and open workflow that others can inspect and reuse for public-benefit purposes.

The output will not be a closed commercial product. It will be published openly for public-interest use.

### How, and where, will you publish the results of your research?

The project will be published openly on GitHub in a public repository. We will publish source code, setup instructions, methodology documentation, attribution details and public releases of the tool/content.

If appropriate, we may also publish a public project page or short report describing the research workflow and findings.

### What open data license will you publish under?

Any derived database or data output incorporating OpenCorporates data will be published under ODbL / share-alike terms where applicable, with appropriate attribution to OpenCorporates.

Source code for the project repository is intended to be released under a separate open-source software licence.

### Will the service or content that you produce be free to use for all or paid for?

The service/content produced by this project will be free to use for all.

### What attribution will you give to OpenCorporates?

We will attribute OpenCorporates clearly in the repository README, documentation and any published outputs.

Where company-level information is displayed, we will link back to the relevant OpenCorporates company page where possible. We will also provide clear source attribution to OpenCorporates as the underlying company data source.
